# SCA — security & compliance analysis collateral

**SCA** in this folder means **security, compliance, and attack-learning analysis deliverables** produced alongside MCP Guardian v2.8.x — not a separate npm package and not traditional dependency SCA (Software Composition Analysis).

The folder bundles **written reports**, **PNG charts**, **reference test scripts**, and a **Next.js + shadcn/ui scaffold** used to prototype dashboards. Core enforcement and learning logic live in the main repo (`src/ai/`, `src/proxy/`).

For **repo-native, reproducible** instant-vs-batch learning numbers, prefer:

- [reports/attack-learning-eval/](../reports/attack-learning-eval/) — `metrics.json`, `figures/`, `summary.md`
- `pnpm eval:attack-learning` (short run) · `pnpm eval:attack-learning:long` · `pnpm eval:attack-learning:charts`

## Next.js scaffold

[package.json](package.json) defines a **standalone Next.js 16** app (`my-project`, private). It is **not** published or required to run MCP Guardian.

```bash
cd sca
pnpm install   # separate from monorepo root
pnpm dev       # http://localhost:3000 — UI shell only
```

The default [app/](app/) layout is a generic shadcn starter. To visualize **live** eval output, wire [ai-learning-dashboard.tsx](ai-learning-dashboard.tsx) to `../reports/attack-learning-eval/metrics.json` or run the repo scripts above.

## Relationship to mcp-guardian core

| Concern | Location |
|---------|----------|
| Instant learning on proxy blocks | `src/ai/instant-attack-learning.ts`, `recordBlockLearningEvent` |
| Batch / debounced learning | `SuggestionEngine`, `GUARDIAN_AI_BLOCK_DEBOUNCE_MS` |
| Reproducible instant vs batch eval | `scripts/lib/attack-learning-eval-core.ts`, `pnpm eval:attack-learning*` |
| Verified metrics & figures | [reports/attack-learning-eval/](../reports/attack-learning-eval/) |
| Operator docs | [docs/AI_LEARNING.md](../docs/AI_LEARNING.md) |

**Honest scope:** SCA reports mix **real Vitest/corpus results** (see root README and `pnpm test`) with **synthetic simulation narratives** (180 min attack sim). When numbers conflict, trust `reports/attack-learning-eval/metrics.json` and CI (`instant-attack-learning` tests) over marketing-style totals in executive summaries.
