MCP Guardian

There is no separate web SPA in v2.5.x. Use the terminal UI for live observability, or call the REST/WebSocket APIs below. A full browser dashboard is planned for v2.6.

Terminal UI (recommended)

pnpm run dogfood    # populate sandbox DB with real proxy traffic
pnpm run tui        # live TUI (WebSocket when proxy/dashboard WS is running)
mcp-guardian doctor
mcp-guardian tui --dashboard-url http://127.0.0.1:4000

WebSocket (live push)

Connect to ws://localhost:4000/ws and send:

{"type":"subscribe","channels":["policy","health","metrics","audit","ai","cost"]}

Enabled when the proxy runs (GUARDIAN_WS_ENABLED defaults to true). Full REST API requires DASHBOARD_ENABLED=true.

REST API (when DASHBOARD_ENABLED=true)

  • GET /api/security — security scans
  • GET /api/cost — cost aggregates
  • GET /api/health — health checks
  • GET /api/instances — instance summary
  • POST /api/policy/suggestions/accept — accept AI suggestion
  • POST /api/policy/suggestions/reject — reject AI suggestion
curl -s http://127.0.0.1:4000/api/instances | jq .
curl -s -X POST http://127.0.0.1:4000/api/policy/suggestions/accept \
  -H 'Content-Type: application/json' \
  -d '{"suggestionId":"baseline-1","ruleName":"rate-limit-burst","source":"baseline","confidence":0.9}'

Environment

  • MCP_GUARDIAN_DB_PATH — SQLite path (default ~/.mcp-guardian/history.db)
  • GUARDIAN_EXPERIMENTAL_AI=true — adaptive learning on proxy/scan/report
  • GUARDIAN_AI_USE_DB_SNAPSHOTS=true — fast learning cycles without live OSV probes
